## Claim

When every project has a standardized structure (graph with deadlines, tasks with status, log with timeline, artifacts with deliverables), the root ike project can traverse all child projects and generate portfolio-level views: Gantt charts across all initiatives, rolled-up status for leadership, resource allocation maps (who is committed to what), and deadline heatmaps. This is the director-level view — not one project, but all projects at once. The root project's graph links to child projects via typed edges. Each child project's graph has deadlines, people, and status. Traversing the root graph at depth 2 would yield: all projects → all deadlines → all people → all blockers. This is how the Wrike portfolio view gets replaced by something the director actually owns and controls.
